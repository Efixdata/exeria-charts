# Data Adapters

Data adapters are the bridge between your chart and external data sources. They handle fetching historical price data and streaming real-time updates.

## What are Data Adapters?

A data adapter is a specialized module that:

- **Fetches historical OHLC data** from a data provider's API
- **Streams real-time price updates** to keep your chart current
- **Manages API credentials securely** — you provide keys, not the package
- **Handles errors gracefully** with automatic retries and reconnection logic

## Why Use Adapters?

✅ **Modularity** — Keep the main chart package lightweight  
✅ **Flexibility** — Use any data source by installing the appropriate adapter  
✅ **Security** — API keys never leave your application  
✅ **Simplicity** — Standard interface makes switching sources easy  
✅ **Extensibility** — Create custom adapters for proprietary data sources

## Quick Start

### 1. Install an adapter

```bash
npm install @efix-data/adapter-binance
```

### 2. Create and configure

```typescript
import { createChart } from "@efixdata/exeria-chart";
import { BinanceAdapter } from "@efix-data/adapter-binance";

const adapter = new BinanceAdapter();
```

### 3. Initialize chart with adapter

```typescript
const chart = createChart({
  container: "#chart",
  dataAdapter: adapter,
});

chart.init();
```

### 4. Load data

```typescript
// Load the most recent 1,000 hourly candles
await chart.loadData("BTCUSDT", {
  interval: "1h",
  limit: 1000,
});

// Subscribe to real-time updates
chart.subscribeToUpdates("BTCUSDT", (tick) => {
  console.log(tick.price ?? tick.c, "at", new Date(tick.stamp));
});
```

You can also bound the range with optional `from` and `to` dates — see [Loading Data](../chart-usage/loading-data#loading-data-with-adapters).

## Creating Custom Adapters

Implement the `DataAdapter` interface exported by `@efixdata/exeria-chart`. Adapters reuse the chart's existing `Candle`, `Tick`, and `Interval` types — there are no separate adapter-specific OHLC shapes.

```typescript
import type {
  Candle,
  DataAdapter,
  LoadDataOptions,
  Tick,
} from "@efixdata/exeria-chart";

class MyAdapter implements DataAdapter {
  async initialize(_config: Record<string, unknown>): Promise<void> {}

  async getHistoricalData(
    symbol: string,
    options: LoadDataOptions,
  ): Promise<Candle[]> {
    // Return candles with stamp, o, h, l, c, v
    return [];
  }

  async getCurrentPrice(symbol: string): Promise<Tick> {
    return { stamp: Date.now(), price: 0 };
  }

  subscribeToUpdates(
    symbol: string,
    callback: (update: Tick) => void,
  ): () => void {
    return () => {};
  }

  async disconnect(): Promise<void> {}
}
```

See the full [API reference](../api-reference/data-adapters) for method details.

## Common Tasks

### Load historical data for multiple intervals

```typescript
const chart = createChart({
  container: "#chart",
  dataAdapter: new BinanceAdapter(),
});
chart.init();

// Daily candles
await chart.loadData("BTCUSDT", {
  interval: "1d",
  limit: 500,
});

// Switch to hourly
await chart.loadData("BTCUSDT", {
  interval: "1h",
  limit: 1000,
});
```

### Load a specific date range

```typescript
await chart.loadData("BTCUSDT", {
  interval: "1d",
  from: new Date("2024-01-01"),
  to: new Date("2024-12-31"),
});
```

### Handle adapter errors

```typescript
try {
  await chart.loadData("BTCUSDT", {
    interval: "1h",
    limit: 1000,
  });
} catch (error) {
  console.error("Failed to load data:", error);
}
```

### Switch symbols

```typescript
chart.unsubscribeFromUpdates();

await chart.loadData("ETHUSDT", {
  interval: "1d",
  limit: 500,
});

chart.subscribeToUpdates("ETHUSDT");
```

## Adapter Lifecycle

1. **Initialize** — Create adapter with configuration
2. **Set** — Provide adapter to chart via `dataAdapter` option or `setDataAdapter()`
3. **Load** — Call `chart.loadData()` to fetch and display historical data
4. **Subscribe** — Call `chart.subscribeToUpdates()` to stream real-time ticks
5. **Disconnect** — Call `adapter.disconnect()` when done

## Best Practices

### API Management

- **Store credentials in environment variables**, not in code
- **Use different keys for development and production**
- **Monitor API usage** to avoid rate limits
- **Don't hardcode symbols** — accept them as parameters

### Error Handling

- **Always wrap `loadData()` in try/catch**
- **Provide feedback** when data loading fails
- **Unsubscribe** before switching symbols or unmounting

### Performance

- **Use `limit`** to control how many candles you fetch
- **Use appropriate intervals** for the visible range
- **Unsubscribe from updates** when not needed
- **Disconnect adapters** when done to free resources

### Security

- **Never log or display API keys**
- **Use HTTPS/WSS connections** (enforced by adapters)
- **Validate user input** before passing to adapters
- **Keep adapters updated** for security patches

## Troubleshooting

### No data appears on chart

1. Verify the symbol is correct and trades on the data source
2. Check that the interval `symbol` is supported by the adapter
3. Wrap `loadData()` in try/catch and log the error
4. Verify API key permissions when a key is required

### Real-time updates not working

1. Check WebSocket connection in browser DevTools
2. Verify the symbol is still valid
3. Look for network or firewall issues
4. Ensure you called `subscribeToUpdates()` after `loadData()`

### API Rate Limits

1. Review the data source's rate limit policies
2. Use `limit` instead of wide date ranges where possible
3. Implement request queuing in high-traffic scenarios
4. Consider upgrading your API tier
