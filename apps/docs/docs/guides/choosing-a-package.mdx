# Choosing a package

Treat the core runtime as the default. Reach for the React wrapper when it saves real product work.

## Choose `@efixdata/exeria-chart` when

- you already have your own application toolbar and menus
- you want the smallest integration surface first
- you need full control over layout, actions, and release cadence

## Choose `@efixdata/exeria-chart-ui-react` when

- you want a faster internal prototype with built-in chart controls
- you are already mounting the runtime chart instance inside React
- you want the toolbar and left-menu layer without rebuilding those controls first

## Practical recommendation

Start with the core chart runtime, prove your data lifecycle and lifecycle cleanup, then add the React UI wrapper if the built-in controls match your product direction.
